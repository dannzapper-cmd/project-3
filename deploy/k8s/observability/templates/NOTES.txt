TRUTHFUL STATUS:
- Metrics: REAL. Prometheus scrapes the AI API /metrics (job=invforge-ai). The
  Grafana dashboard uses only the verified PR-07 metric contract.
- Logs: REAL. Promtail (DaemonSet) ships pod stdout to Loki; the AI API emits
  structured JSON logs. Grafana "Loki" datasource + logs panel.
- Alerts: REAL. 3 rules on real metrics -> AlertManager -> in-cluster webhook
  receiver (alert-webhook-receiver). Confirm with `kubectl logs`.
- Traces: PENDING. Tempo + OTel Collector are deployed as OTLP receivers but the
  AI API is NOT OTEL-instrumented yet, so no traces flow. Enabling traces is
  deferred (see docs/runbooks/otel-tracing.md). The Tempo panel says so.

NOTE on NetworkPolicy: PR-11A ships structural NetworkPolicy in invforge-ai.
kindnet does not enforce it, so cross-namespace scraping works on default kind.
If you enable Calico, allow ingress from {{ .Values.namespace }} to the AI API.

Redis inference caching is intentionally NOT here (no cache integration in the
API). Mimir and OpenMetadata are intentionally excluded (see ADR 003).
